# んだばい 予約＋決済バックエンド（GAS）デプロイ手順

`shared/booking.js` の予約フォームから **Stripe Checkout（カード決済）** へ進み、
**決済が成功した時点で** Google スプレッドシートへ記録・Google カレンダー登録・
お客様／管理者への確認メール送信を行います。さらに月次で売上を集計（コミッション10%、決済済みのみ）します。

## 決済フロー（全体像）

```
予約フォーム送信
  └ 隠しフォームPOST(action=checkout) ─▶ GAS doPost
                                          └ Stripe Checkout セッション作成（秘密鍵はScript Properties）
                                          └ 予約を「決済待ち」で先行記録
        ◀─ HTMLでStripeへリダイレクト ────┘
   Stripe Checkout（カード入力・決済）
        ── 成功 ─▶ success_url = GAS(action=confirm) ─▶ 決済検証→「決済済み」更新＋カレンダー＋メール→完了画面
        ── 取消 ─▶ 予約ページ(?canceled=1)（フォームに戻りメッセージ表示）
```

- CORS回避のため、フロントは fetch ではなく **トップレベル遷移（隠しフォームPOST）** で GAS を呼びます。
- 決済の最終確定は **成功リダイレクト時の `action=confirm`** で行います（冪等）。
- 取りこぼし救済として `reconcilePendingPayments()`（「決済待ち」をStripeで再確認）を任意でトリガー実行できます。

## スプレッドシート構成（初回送信時に自動作成）

| シート | 内容 |
|--------|------|
| 田村市予約 | program=tamura の予約 |
| 葛尾村予約 | program=katsurao の予約 |
| 売上集計 | 月 / 田村市売上 / 葛尾村売上 / 合計 / コミッション(10%) |

予約シートの列: 受付日時・プログラム・コース・人数・希望日・オプション・合計金額・氏名・メール・電話・健康状態・備考・ステータス・StripeセッションID
（ステータス: `決済待ち` → 決済成功で `決済済み`。売上集計は `決済済み` のみ対象）

## Stripe 決済の設定（必須）

1. **Stripe アカウントでキーを取得**
   ダッシュボード（テストモード）→「開発者 → APIキー」から **シークレットキー `sk_test_...`** を取得。
   （Checkout リダイレクト方式のため、公開可能キー `pk_test_...` はフロントには不要です）

2. **GAS にシークレットキーを保存（コードに直書きしない）**
   Apps Script エディタ →「⚙ プロジェクトの設定 → スクリプト プロパティ → プロパティを追加」
   - プロパティ: `STRIPE_SECRET_KEY`
   - 値: `sk_test_xxxxxxxx`（本番移行時に `sk_live_...` へ差し替え）

3. **ウェブアプリを（再）デプロイ**
   コード更新後は「デプロイ → デプロイを管理 → 編集 → 新バージョン」で再デプロイ。
   `success_url` は GAS が自身のURL（`action=confirm`）を使うため、追加設定は不要です。

4. **テスト決済**
   テストカード番号 `4242 4242 4242 4242` / 有効期限=未来の日付 / CVC=任意3桁 / 郵便番号=任意。
   決済後、完了画面が表示され、シートが `決済済み` になり、確認メールが届けば成功です。

5. **本番移行**
   `STRIPE_SECRET_KEY` を本番キー `sk_live_...` に差し替え、再デプロイ。Stripe側で本番有効化（事業者情報・口座）を完了しておくこと。

## デプロイ手順

1. **スプレッドシート作成（推奨：コンテナバインド）**
   Google スプレッドシートを新規作成し、「拡張機能 → Apps Script」でスクリプトエディタを開く。
   この方法なら `SPREADSHEET_ID` は空のままでOK（`getActiveSpreadsheet()` を使用）。
   ※ スタンドアロンのスクリプトにする場合は、`Code.gs` の `SPREADSHEET_ID` にスプレッドシートIDを設定。

2. **コードを貼り付け & 設定**
   `Code.gs` の内容を貼り付け、先頭の CONFIG を編集：
   - `ADMIN_EMAIL` … 管理者通知先（既定 `info@ndanda.net`）
   - `CALENDAR_ID` … 空ならデフォルトカレンダー。専用カレンダーを使う場合はそのIDを設定。

3. **ウェブアプリとしてデプロイ**
   「デプロイ → 新しいデプロイ → 種類: ウェブアプリ」
   - 実行ユーザー: 自分
   - アクセスできるユーザー: **全員（匿名含む）**
   「デプロイ」→ 初回は権限承認（スプレッドシート・カレンダー・メール送信）。

4. **ウェブアプリURLをコピー → フロントに設定**
   発行されたURLを `shared/booking.js` の `const GAS_ENDPOINT = '';` に貼り付ける。
   （空のままだとフォームは「デモモード」で動作し、送信は記録されません）

5. **月次売上集計トリガーの登録（任意）**
   スクリプトエディタで関数 `setupMonthlyTrigger` を一度手動実行すると、
   毎月1日 2時に `updateMonthlySales` が走り、売上集計シートを更新します。

6. **動作確認**
   各プログラムページのフォームからテスト送信し、
   - 対応するシート（田村市/葛尾村）に行が追加される
   - カレンダーに「【予約】…」イベントが作成される
   - お客様宛・管理者宛の両メールが届く
   ことを確認。

## 受信データ（決済フロー / shared/booking.js が送信）

決済時はフォーム(`application/x-www-form-urlencoded`)で以下を `action=checkout` とともにPOSTします
（`e.parameter` で受信）。

| フィールド | 例 | 用途 |
|---|---|---|
| `action` | `checkout` | 処理分岐 |
| `program` | `tamura` / `katsurao` | シート振り分け |
| `course` | `1泊2日` | コース名 |
| `price` | `49000` | コース単価（円・税込） |
| `guests` | `2` | 人数 |
| `date` | `2026-06-18` | 希望日 |
| `options` | `リンパマッサージ、ヨガ` | オプション名（区切り文字列） |
| `optionsTotal` | `20000` | オプション合計（1予約あたり） |
| `total` | `118000` | 合計（`price × guests + optionsTotal`） |
| `name` / `email` / `phone` | — | 連絡先 |
| `health` / `note` | — | 健康状態・備考 |
| `page_url` | `https://…/tamura/` | 取消時の戻り先 |

- Stripe明細: 「コース × 人数」＋「オプション（あれば1明細）」。通貨 `jpy`（ゼロ小数通貨＝そのままの金額）。
- 旧クライアント（JSONボディ直接POST・決済なし）も後方互換で受理します（`legacyRecord`）。

## トラブルシューティング

- **初回に承認画面** … 「詳細 → 安全ではないページに移動 → 許可」で承認。
- **確認メールが届かない** … 迷惑メールフォルダを確認。`MailApp` の1日あたり送信上限にも注意。
- **カレンダーに入らない** … `CALENDAR_ID` の指定、または希望日(date)の形式(yyyy-mm-dd)を確認。エラーは実行ログに出力（予約記録自体は継続）。
- **シートが無い** … 初回送信時に自動作成されます。
