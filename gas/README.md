# んだばい 予約管理バックエンド（GAS）デプロイ手順

`shared/booking.js` の予約フォームから送信された内容を Google スプレッドシートに記録し、
Google カレンダーに予定を作成、お客様・管理者へ確認メールを自動送信します。
さらに月次で売上を集計（コミッション10%）します。

## スプレッドシート構成（初回送信時に自動作成）

| シート | 内容 |
|--------|------|
| 田村市予約 | program=tamura の予約 |
| 葛尾村予約 | program=katsurao の予約 |
| 売上集計 | 月 / 田村市売上 / 葛尾村売上 / 合計 / コミッション(10%) |

予約シートの列: 受付日時・プログラム・コース・人数・希望日・オプション・合計金額・氏名・メール・電話・健康状態・備考・ステータス

## デプロイ手順

1. **スプレッドシート作成（推奨：コンテナバインド）**
   Google スプレッドシートを新規作成し、「拡張機能 → Apps Script」でスクリプトエディタを開く。
   この方法なら `SPREADSHEET_ID` は空のままでOK（`getActiveSpreadsheet()` を使用）。
   ※ スタンドアロンのスクリプトにする場合は、`Code.gs` の `SPREADSHEET_ID` にスプレッドシートIDを設定。

2. **コードを貼り付け & 設定**
   `Code.gs` の内容を貼り付け、先頭の CONFIG を編集：
   - `ADMIN_EMAIL` … 管理者通知先（既定 `info@ndanda.net`）
   - `CALENDAR_ID` … 空ならデフォルトカレンダー。専用カレンダーを使う場合はそのIDを設定。

3. **ウェブアプリとしてデプロイ**
   「デプロイ → 新しいデプロイ → 種類: ウェブアプリ」
   - 実行ユーザー: 自分
   - アクセスできるユーザー: **全員（匿名含む）**
   「デプロイ」→ 初回は権限承認（スプレッドシート・カレンダー・メール送信）。

4. **ウェブアプリURLをコピー → フロントに設定**
   発行されたURLを `shared/booking.js` の `const GAS_ENDPOINT = '';` に貼り付ける。
   （空のままだとフォームは「デモモード」で動作し、送信は記録されません）

5. **月次売上集計トリガーの登録（任意）**
   スクリプトエディタで関数 `setupMonthlyTrigger` を一度手動実行すると、
   毎月1日 2時に `updateMonthlySales` が走り、売上集計シートを更新します。

6. **動作確認**
   各プログラムページのフォームからテスト送信し、
   - 対応するシート（田村市/葛尾村）に行が追加される
   - カレンダーに「【予約】…」イベントが作成される
   - お客様宛・管理者宛の両メールが届く
   ことを確認。

## 受信データ（JSON契約 / shared/booking.js が送信）

```json
{
  "program": "tamura | katsurao",
  "course": "1泊2日",
  "price": 49000,
  "guests": 2,
  "date": "2026-06-18",
  "options": ["リンパマッサージ", "ヨガ"],
  "optionsTotal": 20000,
  "total": 118000,
  "name": "山田太郎",
  "email": "yamada@example.com",
  "phone": "090-0000-0000",
  "health": "特になし",
  "note": "",
  "timestamp": "2026-05-02T16:00:00.000Z"
}
```

- 金額計算: `total = price × guests + optionsTotal`（オプションは1予約あたり）
- `program` でシートを振り分け（tamura→田村市予約 / katsurao→葛尾村予約）

## トラブルシューティング

- **初回に承認画面** … 「詳細 → 安全ではないページに移動 → 許可」で承認。
- **確認メールが届かない** … 迷惑メールフォルダを確認。`MailApp` の1日あたり送信上限にも注意。
- **カレンダーに入らない** … `CALENDAR_ID` の指定、または希望日(date)の形式(yyyy-mm-dd)を確認。エラーは実行ログに出力（予約記録自体は継続）。
- **シートが無い** … 初回送信時に自動作成されます。
